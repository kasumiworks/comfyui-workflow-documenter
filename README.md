# ComfyUI Workflow Documenter

**Drop a ComfyUI workflow — get clean, human-readable documentation. Free, no install, 100% local.**

👉 **Use it here: https://kasumiworks.github.io/comfyui-workflow-documenter/**

## What it does

Drag & drop a workflow `.json` (UI export or API format) — or a **PNG generated by ComfyUI** (it reads the embedded metadata) — and you instantly get:

- ✅ **Custom node pack check** — see exactly which packs you need to install, or a "runs on a clean install" badge if it's 100% core nodes
- 📦 **Required model files** — checkpoints, LoRAs (with strengths, in load order), VAE, upscalers, ControlNets
- 📁 **Local models check** — point it at your ComfyUI `models` folder (read-only, in-browser) and see which files you already have: exact match, close variant (fp16 ⇄ fp8, different quants), or missing. Chromium-based browsers only.
- ⚙️ **Generation settings** — resolution, steps, CFG, sampler, scheduler, seed, multi-pass breakdown
- ✍️ **Prompts** — positive/negative, traced through the actual node graph (not guessed). Shown first, right after the overview.
- 🔀 **Pipeline overview** — the main path from loader to output
- 📋 **Full node reference table**

Then export it two ways:

- **Copy Markdown / Download .md** — full docs with tables, ready for a GitHub README or product page
- 🎨 **Copy for Civitai** — same info reformatted for Civitai's editor (which mangles `#` headers and tables): bold headings, bullet lists, prompts at the top, post-sized

## Privacy

Everything runs in your browser. **Your workflows and images never leave your machine.** No upload, no server, no account, no analytics. You can download `index.html` and run it offline.

The local models check uses the File System Access API in **read-only** mode — it only reads filenames to compare against the workflow, scans entirely in your browser, and uploads nothing.

## Why

Sharing workflows is easy. Making them *understandable* is not. Most "here's my workflow" posts end with 50 comments asking "what custom nodes do I need?" and "which model is this?". This tool answers those questions automatically.

## Run locally / offline

Download `index.html`, double-click it. That's it — it's a single self-contained file with zero dependencies.
